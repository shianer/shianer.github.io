  • 基础知识类型：一般不会有，有也是Web签到题（不结合其他复杂知识的话）

    查看网页源代码

    fn+f12(微软，火狐旧版可开启开发者工具）在我的电脑上f12只能打开联想电脑管家，或Ctrl+U（这个一般浏览器都可view-source:)

    (大概率在注释里有点东西，flag啊，hint之类有关的东西。）前端代码要分析html和js代码，修改使之满足要求，或利用控制台浏览器解析混淆代码

    关于编码然后想补一点的浅浅东西：

    ctf常见代码：加密与编码的区别（加密是利用数学方法将邮件转换为不可读格式从而达到保护数据的目的的一门科学，它的样式已经改变，必须要有密钥才能解开。解码，是编码的逆过程。从概念上可以看出，编码只是表现形式的转换，没有保密的作用，因为编码和解码的算法是公开的，只要知道是什么编码的内容，任何人都可以轻松地解码。网上有一句形象生动的说法：加密就类似于整容，现在的样子和原来的样子已经千差万别，要想认出他/她，只能通过身体的其他的特有的特征（密钥）；编码类似于化妆，你依然可以看到她/他原来的样子（方法就是卸妆）。我管蠡窥测，目前只了解的web题大多数只是有关编码的，没见过古密，以后学到了再补充。）

    传送门加解码

    1.Base32加密：（还有16/64等链接有）
    例：KRUGS4ZANFZSAYLOEBSXQYLNOBWGK===
    通常在其后会有三个等号，比较好判断，当然没有‘=’也可以解得出来 0.0

    2.UUencode编码
    例： 2Z+^9Y+BJYIBOY+BJY+Z+Y:V0
    往往会出现+，-，；等比较奇怪的符号

    3.URL编码（UrlEncode）：
    例：%e8%bf%99%e6%98%af%e4%b8%aa%e4%be%8b%e5%ad%90
    因为出现了%因此也比较容易认出来

    4.unicode编码
    例：\u0054\u0068\u0069

    每一行都有’\u’

    5.HTML编码
    例子：以“&#100；&#101；”类似为特征

    6.unescape
    例子：%u8FD9%u662F%u4E2A%u4F8B%u5B50
    %u 开头也很好辨认

    7.16进制
    秘文里出现不超过F的字母和数字，则有很大可能就是16进制，不多做解释。

    8.BCD码.(Binary-Coded Decimal)
    二进码十进数，是一种二进制的数字编码形式，形式如：XXXXYYYY
    通常是指8421码，就是将十进制的数以8421的形式展开成二进制，大家知道十进制是0～9十个数组成，这十个数每个数都有自己的8421码。

    9.还有一些代码混淆(是将计算机程序的代码，转换成一种功能上等价，但是难于阅读和理解的形式的行为。)

    1. asp混淆加密
    2. php混淆加密
    3. css（除了html的超文本标记语言外的网页页面效果）/js（一些元素如动态效果）混淆加密
    4. VBScript.Encode混淆加密 针对asp代码进行的一种加密混淆操作
    5. ppencode 把Perl代码转换成只有英文字母的字符串
    6. rrencode 类似jjencodeuby，把代码全部转换成符号
    7. jjencode/aaencode 将 JS 代码转换成只有符号的字符串/JS 代码转换成常用的网络表情，也就是我们说的颜文字 js 加密
    8. JSfuck 只用 6 个字符[ ]( ) ! + 来编写 JavaScript 程序。此外，JSFuck可用于绕过恶意代码检测，且可以被用于跨站脚本攻击。
    9. jother 利用少量特定字符（javascript用 ! [] + - * 表示0~1000的数字）构造精简的匿名函数对与字符串的编码方式，console将那堆解码的东西复制过去 按回车即可解码
    10. brainfuck编程语言 八种符号(> < + – . , [ ])

    http请求（b-6~25）有关内容，

    1.发送http请求（hackbar,写脚本），

    2.不常见类型的请求方式（脚本），

    3.HTTP头有关题目：查看和修改http头。常见题，不同类型有不同方法，但基本离不开抓包，熟悉一些常见的请求头格式，作用。

    4.查看响应头：可能会有hint,关键的信息，flag基本别想了。

    (1)修改请求头，伪造cookie：

    只有客户端才能设置Cookie，服务端若想让客户端增加一个Cookie项，需要在应答时，在Http头部中，通过使用Set-Cookie，将要设置的Cookie项发送给客户端。这样客户端，在下次访问时，会带上该Cookie项。

    （2）Cookie：变量，独立于语言存在，也就是说，不论是PHP还是JSP（亦或是Java、.Net等其他语言）设置的Cookie，其本质都是一样滴，客户端脚本Javascript都能读取到。

    set-cookie是一个函数，由服务器发送一个HTTP-cookie到浏览器。

    （3）XFF:

    XFF是header请求头中的一个参数。是用来识别通过HTTP代理或负载均衡方式连接到Web服务器的客户端最原始的IP地址的HTTP请求头字段。代表了HTTP的请求端真实的IP。修改XFF头的信息，即可绕过服务器的过滤。

    X-Forwarded-For: 127.0.0.1 -> X-Forwarded-For: 192.168.1.1

    (4)Referer:

    作为防盗链 ： 将这个http请求发给服务器后，如果服务器要求必须是某个地址或者某几个地址才能访问，而你发送的referer不符合他的要求，就会拦截或者跳转到他要求的地址，然后再通过这个地址进行访问。

    防止恶意请求：比如静态请求是*.html结尾的，动态请求是*.shtml，那么所有的*.shtml请求，必须 Referer为我自己的网站才可以访问，这就是Referer的作用。

    空referer:直接在浏览器的地址栏中输入一个资源的URL地址，那么这种请求是不会包含 Referer 字段的，因为这是一个“凭空产生”的 HTTP 请求，并不是从一个地方链接过去的。

    **修改页面元素**啊，就是要恶补基础知识啦~~~

    修改一个网页中的内容，这个技术不会也不能修改服务器动态传递的数据，也不能修改服务器中的数据，更不能将修改后的数据传回服务器（非用户输入的情况下），只能修改已经被加载完毕的静态数据，只能修改云端而已，刷新就没了。基于Chrome内核，部分操作可能会不同，但是思路是相同的。毕竟在不同的浏览器中，调试工具可能会有不同的名字，甚至是有不同的开启方法。哈哈太浅了，总不可能藏在图层下，如此也在element中有吧。。。下次再修正

    • sql注入

    就是对SQL语言要弄明白，根据输入不同数据网页的反应对后台语句的判断可以得出来，自动化工具sqlmap(kail环境下，明天试试安一个，感觉学习新知识都没有让我安装东西难啊。。)，在sql漏洞学习完可以试试!但是不知道比赛时能不能用，SO还是好好学sql手工注入扎实基础。

    sqlmap可以与burp suite ,google(虚拟机中尝试代理),结合使用.

    • 可以检测页面中get,post参数,cookie,http头等.
    • 可以实现数据榨取
    • 可以实现文件系统的访问
    • 可以实现操作命令的执行
    • 还可以对xss漏洞进行检测

    sqlmap练习 （DVWA）支持5种漏洞检测类型:（之后在写sqlilabs的blog中会再拓展，不然我估计又会忘记啦）

    • 基于布尔的盲注检测 (如果一个url的地址为xxxx.php?id=1,那么我们可以尝试下的加上 and 1=1(和没加and1=1结果保持一致) 和 and 1=2(和不加and1=2结果不一致),则我们基本可以确定是存在布尔注入的. )

    • 基于时间的盲注检测(和基于布尔的检测有些类似.通过mysql的 sleep(int)) 来观察浏览器的响应是否等待了你设定的那个值 如果等待了,则表示执行了sleep,则基本确定是存在sql注入的

    • 基于错误的检测 (组合查询语句,看是否报错(在服务器没有抑制报错信息的前提下),如果报错 则证明我们组合的查询语句特定的字符被应用了,如果不报错,则我们输入的特殊字符很可能被服务器给过滤掉(也可能是抑制了错误输出.))

    • 基于union联合查询的检测(适用于如果某个web项目对查询结果只展示一条而我们需要多条的时候 则使用union联合查询搭配concat还进行获取更多的信息)

    • 基于堆叠查询的检测(首先看服务器支不支持多语句查询,一般服务器sql语句都是写死的,某些特定的地方用占位符来接受用户输入的变量,这样即使我们加and 也只能执行select(也不一定select,主要看应用场景,总之就是服务端写了什么,你就能执行什么)查询语句,如果能插入分号;则我们后面可以自己组合update,insert,delete等语句来进行进一步操作

      • 绕过waf(Web应用防火墙是通过执行一系列针对HTTP/HTTPS的安全策略来专门为Web提供保护的一种产品。)*

        必要条件：1.熟练掌握mysql函数和语法使用方法 2，深入了解中间件运行处理机制 3，了解WAF防护处理原理和方法

      要求输入较长的文本，但对其长度进行限制。可以在BrupSuite抓包改包绕过，也可以在F12中改源码。

      修改元素中HTML maxlength 属性: 规定 <input> 元素中允许的最大字符数
