webshell

webshell就是以asp、php、jsp或者cgi等网页文件形式存在的一种代码执行环境，主要用于网站管理、服务器管理、权限管理等操作。使用方法简单，只需上传一个代码文件，通过网址访问，便可进行很多日常操作，极大地方便了使用者对网站和服务器的管理。正因如此，也有小部分人将代码修改后当作后门程序使用，以达到控制网站服务器的目的。

第十一关：黑名单的最后一题

第十二关：白名单

这里乍一看和第一题很像，但是它又不存在客户端的验证漏洞，是属于比较安全的白名单。

这里只允许上传“jpg,png,gif”三种类型的文件，但是上传路径是可以控制的，可以使用%00来进行截断（只用于php版本低于5.3的，并且在小皮面板中要把php.ini中的magic_quotes_gpc修改为off状态，记得电脑要重启一遍）。这里对上传的文件进行文件名重构，那么我们就可以利用重构进行绕过。

这里的原理：$_GET[‘save_path’]函数是文件最终存储路径。代码里面拼接了一些随机字符，我们可以上传1.jpg，而在 save_path这个最终保存路径函数下面保存为1.php并加上%00来隔断后面的随机字符，这样实际存储到服务器上面路径上就是1.php的后缀了。

这里利用的是00截断。即move_uploaded_file函数的底层实现类似于C语言，遇到0x00会截断

截断条件：
1、php版本小于5.3.4
2、php.ini的magic_quotes_gpc为OFF状态

0x：16进制表示
00：表示0
0x00：就是代表16进制的0，在ASCII码里代表null。

有的函数在处理这个字符时，会当做结束符

%00 和 00 是一样的，只是在get提交时，经过url编码后，00就成了%00

00截断是十六进制的00，当操作系统在读文件名的时候在读到00时他认为这个文件结束了，后面的一律忽视。于是我们上传一个木马文件，改为jpg格式，在保存路径里面添加一个php文件后面加%00，这样就绕过上传然后又上传保存的变成php格式的。

第十四关：图片马加文件包含漏洞，使用文件包含才能解析木马的运行。

文件头格式:

补充知识：
1.Png图片文件包括8字节：89 50 4E 47 0D 0A 1A 0A。即为 .PNG。
2.Jpg图片文件包括2字节：FF D8。
3.Gif图片文件包括6字节：47 49 46 38 39|37 61 。即为 GIF89(7)a。
4.Bmp图片文件包括2字节：42 4D。即为 BM。

制作过程有两种：

一种用命令行copy 14.jpg/a + 14.php/b n14.jpg

另一种用notepad打开jpg文件添加上php文件。

这里我在试jpg格式一直出不来，其他两种顺利通过。

第十七题：二次渲染绕过

这一题用到了imagecreatefromjpeg（）函数

原理：二次渲染是由GIF文件或URL创建一个新图像。成功则返回–图像标识符/图像资源，失败返回false，反正就是导致图片马丢失，上传失败。

按先前的方法发现可以上传但是没办法和包含漏洞配合解析，经过二次渲染图片中的PHP代码被删掉了。而我们要做的就是比较原图与他修改的图片，看看哪个部分没被改过，把代码放到那里，配合漏洞。用winhex修改后即可，上传，一遍遍试试合适的位置，但是一般好像可以用python程序。

第十八关：条件竞争

介绍：

条件竞争漏洞是一种服务器端的漏洞，是由于开发者设计应用程序并发处理时操作逻辑不合理而造成。当应用面临高并发的请求时未能同步好所有请求，导致请求与请求之间产生等待时出现逻辑缺陷。该漏洞一般出现在与数据库系统频繁交互的位置，例如金额同步、支付等较敏感操作处。另外条件竞争漏洞也会出现在其他位置，例如文件的操作处理等。
首先将文件上传到服务器，然后检测文件后缀名，如果不符合条件，就删掉，典型的“引狼入室”。

我们看代码他是先将图片上传上去，才开始进行判断后缀名、二次渲染。如果我们在上传上去的一瞬间访问这个文件，那他就不能对这个文件删除、二次渲染。这就相当于我们打开了一个文件，然后再去删除这个文件，就会提示这个文件在另一程序中打开无法删除。

解题步骤：

用测试器来不断在文件上传到删除过程中访问文件，用py执行在浏览器中打开。

可以看到上传该文件的数据包不停地在进行重放。

在BP攻击的同时我们也要运行python脚本，目的就是不停地访问shell.php知道成功访问到为止。当出现OK说明访问到了该文件，那么shell.php应该也创建成功了，用蚁剑连一下试试。

第十九关：

从源码来看，服务器用白名单检查了文件后缀和文件大小，是否存在，上传后并进行重命名。

解题方法：

1.所以我们上传图片马并在文件未被重命名前访问他。让图片马执行还需要配合其他漏洞：文件包含漏洞，Apache解析漏洞等

首先制作，再抓包无限重放，然后修改脚本不断通过文件包含访问图片马。然后通过脚本中添加页面的标识词来判断是否上传成功。

同样的这也属于条件竞争的一种，只不过文件的形式不同而已。

2.其实可以直接上传图片马，因为页面会回显改名后的图片马的位置，直接文件包含也能生成shell.php。

3.值得注意的是这里也可以将php文件后缀名更改为.php.7z，因为白名单中允许上传.7z的文件，但是apache又不能解析这个格式，所以会把该文件当php的格式解析。

这里注意有一个错误要修改：

第二十关：

看源码我们发现，服务器端定义了一系列黑名单，而move_uploaded_file()函数的img_path是由参数save_name控制的，可以在save_name利用%00截断’.php%00.jpg’，也可以用/.绕过，同时没有大小写限制也可以对保存名用PHP后缀绕过。

第二十一关：

$is_upload = false;$msg = null;if(!empty($_FILES[‘upload_file’])){
//检查MIME
$allow_type = array(‘image/jpeg’,’image/png’,’image/gif’);
if(!in_array($_FILES[‘upload_file’][‘type’],$allow_type)){
$msg = “禁止上传该类型文件!”;
}else{
//检查文件名
$file = empty($_POST[‘save_name’]) ? $_FILES[‘upload_file’][‘name’] : $_POST[‘save_name’];
if (!is_array($file)) {
$file = explode(‘.’, strtolower($file));
}

    $ext = end($file);
    $allow_suffix = array('jpg','png','gif');
    if (!in_array($ext, $allow_suffix)) {
        $msg = "禁止上传该后缀文件!";
    }else{
        $file_name = reset($file) . '.' . $file[count($file) - 1];
        $temp_file = $_FILES['upload_file']['tmp_name'];
        $img_path = UPLOAD_PATH . '/' .$file_name;
        if (move_uploaded_file($temp_file, $img_path)) {
            $msg = "文件上传成功！";
            $is_upload = true;
        } else {
            $msg = "文件上传失败！";
        }
    }
}}else{
$msg = "请选择要上传的文件！";}

检测文件后缀名，属于白名单，服务器端先检查了MIME类型，然后判断save_name参数是否为空，空就把文件本来的名字赋值给$file，否则就把save_name的参数值赋值给他。接着判断$file是否为数组，不是数组拆成数组，最后一个值(end函数就是取数组最后一个的值)同白名单做比较，符合jpg、png、gif中的一种就允许上传了。在允许上传之后还要把数组的值拼接在一起对文件进行重命名。

所以我们可以构造save_name[0]=cs.php/ save_name[1]置为空 save_name[2]=jpg(一个白名单的合法后缀)。

这样的话，reset($file)取的是数组的第一个元素即cs.php/，然后接了一个’.’符号，之后又将数组最后一个元素内容拼接到一起。

可能有的人会疑问数组最后一个值不是jpg吗？其实当我们只设置了两个数组元素的时候，数组的元素个数就只有两个了。

既然一共只有两个元素，这里就是$file[2-1]也就是$file[1]。因此拼接的就是空的，最终得到的文件名就是cs.php/.。

对于像cs.php/.这样的文件路径，move_uploaded_file()函数会忽略掉文件末尾的/.。如此一来我们上传到服务器的文件还是被重命名为了php后缀。
